МЕТА

Ознайомитися із процесом завантаження, встановлення та налаштування Android Studio, офіційного середовища розробки для створення застосунків для Android. Вивчити основні елементи інтерфейсу Android Studio та налаштування для роботи з емуляторами Android. Ознайомитися із базовими кроками для створення мінімального Android застосунку. Вивчити основні елементи структури Android-застосунку та процес компіляції та запуску на емуляторі або фізичному пристрої. Навчитися створювати GitHub репозиторій. Вивчити основи роботи з системою контролю версій Git. Освоїти процес публікації коду Android-застосунку на GitHub.

ХІД РОБОТИ
	1. Встановлення Android Studio
	1.1. Завантаження Android Studio
	1.1.1. Відвідайте офіційний вебсайт Android Studio та завантажте останню версію Android Studio, яка відповідає вашій операційній системі (Windows, macOS або Linux).
	1.1.2. Ознайомтеся з системними вимогами Android Studio. Перевірте наявність на вашому комп’ютері необхідних ресурсів (процесор, оперативна пам'ять, вільне місце на диску).
	1.2. Встановлення Android Studio
	1.2.1. Запустіть інсталяційний файл Android Studio.
	1.2.2. Слідуйте покроковим інструкціям майстра встановлення: - Погодьтеся з умовами ліцензії. - Виберіть компоненти для встановлення (Android SDK, Android Virtual Device (AVD) Manager). - Вкажіть директорію для встановлення програми та SDK.
	1.2.3. Завершіть встановлення, дочекайтеся завантаження та налаштування Android SDK.
	1.3. Налаштування Android Studio після встановлення
	1.3.1. Після першого запуску Android Studio майстер налаштувань запропонує налаштувати параметри: - Виберіть тип інтерфейсу (стандартний або темний). - Перевірте оновлення компонентів Android SDK. - Налаштуйте емулятори Android (Android Virtual Device Manager).

	1.3.2. Створіть новий проект, щоб перевірити, чи працюють усі компоненти належним чином: - Виберіть тип проекту (Empty Activity). - Встановіть мінімальну версію Android SDK для свого проекту.
	1.4. Налаштування емулятора Android
	1.4.1. Запустіть Android Virtual Device Manager для створення віртуального пристрою Android: - Виберіть тип пристрою (телефон, планшет або інший). - Виберіть версію Android для емулятора. - Налаштуйте інші параметри (розмір екрану, обсяг пам’яті).
	1.4.2. Запустіть емулятор і перевірте його роботу.

	2. Створення мінімального Android застосунку
	2.1. Створення нового Android проєкту
	2.1.1. Відкрийте Android Studio та натисніть на кнопку "Start a new Android Studio project".
	2.1.2. Виберіть шаблон для створення мінімального проєкту: - Оберіть Empty Activity як шаблон для застосунку. - Натисніть Next.
	2.1.3. Вкажіть основну інформацію про застосунок: - Назва проєкту: Введіть бажану назву застосунку. - Ім'я пакету: Зазвичай складається з доменного імені у зворотному порядку (наприклад, com.example.myapplication). - Місце збереження проєкту: Виберіть директорію для збереження файлів проєкту. - Мова програмування: Оберіть Java або Kotlin (залежно від пріоритетів). - Мінімальна версія SDK: Виберіть мінімальну версію Android, яка підтримуватиме ваш застосунок.
	2.1.4. Натисніть Finish для завершення створення проєкту.
	2.2. Ознайомлення зі структурою проєкту
	2.2.1. Файли проєкту: - MainActivity.java/MainActivity.kt: Головний клас вашого застосунку, що відповідає за запуск Activity. - res/layout/activity_main.xml: Файл розмітки інтерфейсу користувача для головного екрану застосунку. - AndroidManifest.xml: Файл конфігурації застосунку, що містить інформацію про компоненти (Activity, Service, BroadcastReceiver тощо).
	2.2. Вивчіть кожен із файлів: - У файлі MainActivity.java/MainActivity.kt знайдіть метод onCreate(), який викликається при створенні Activity. - У файлі activity_main.xml переконайтеся, що існує базовий елемент (наприклад, TextView).
	2.3. Налаштування емулятора або підключення фізичного пристрою
	2.3.1. Якщо ще не створено емулятор, скористайтеся Android Virtual Device (AVD) Manager, щоб налаштувати віртуальний пристрій.
	2.3.2. Підключіть фізичний пристрій (телефон або планшет) через USB і увімкніть режим налагодження (Developer Mode).
	2.4. Запуск мінімального застосунку
	2.4.1. Натисніть кнопку Run або виберіть Run -> Run 'app' для компіляції та запуску застосунку.
	2.4.2. Виберіть емулятор або фізичний пристрій для запуску.
	2.4.3. Після успішної компіляції застосунок запуститься на вибраному пристрої, де на екрані буде відображено простий інтерфейс із елементом TextView.
	2.2.5. Внесення змін до інтерфейсу
	2.5.1. Відкрийте файл activity_main.xml у редакторі та додайте нові елементи інтерфейсу, наприклад, кнопку (Button).
	2.5.2. Вкажіть властивості для нових елементів у XML-коді: - Встановіть текст кнопки за допомогою атрибуту android:text="Натисніть мене". - Встановіть розміри кнопки, використовуючи android:layout_width та android:layout_height. Програмний код у додатку А.
	2.5.3. Запустіть застосунок ще раз, щоб перевірити зміни.
	2.6. Взаємодія з інтерфейсом
	2.6.1. У файлі MainActivity.java/MainActivity.kt додайте обробку натискання на кнопку: - Знайдіть кнопку за її ідентифікатором за допомогою методу findViewById(). - Додайте обробник натискання за допомогою методу setOnClickListener().
	2.6.2.	 Змініть текст елемента TextView при натисканні кнопки.
	2.6.3. Перевірте роботу застосунку, запустивши його ще раз.

3. Створення репозиторію та публікація коду застосунку
3.1. Налаштування Git в Android Studio
3.1.1. Відкрийте ваш Android-застосунок в Android Studio.
3.1.2. Переконайтеся, що Git встановлено на вашому комп'ютері. Якщо Git не встановлено, завантажте та встановіть його з офіційного сайту Git.
3.3. Налаштуйте Git в Android Studio: - У меню Android Studio виберіть File -> Settings (Preferences на macOS) -> Version Control -> Git. - У полі Path to Git executable перевірте правильність шляху до виконуваного файлу Git.
3.4. Ініціалізуйте Git у вашому проєкті: - Відкрийте VCS -> Enable Version Control Integration. - У випадаючому меню оберіть Git.
3.2. Створення нового GitHub репозиторію
3.2.1. Перейдіть на GitHub та увійдіть до свого акаунту.
3.2.2. Створіть новий репозиторій: - Натисніть кнопку New repository у верхньому правому куті сторінки. - Введіть назву репозиторію, наприклад, MyFirstApp. - За потреби, додайте опис репозиторію. - Виберіть тип репозиторію: Public (відкритий для всіх) або Private (доступний лише для вас та вибраних користувачів). - Натисніть кнопку Create repository
3.2.3. Скопіюйте URL вашого нового репозиторію для подальшого використання.
3.3. Публікація коду застосунку на GitHub
3.3.1. Поверніться в Android Studio та додайте проєкт до GitHub: - Відкрийте VCS -> Git -> Add для додавання всіх файлів проєкту в індекс Git.
3.3.2. Створіть перший коміт: - Виберіть VCS -> Commit або натисніть клавіші Ctrl+K (Cmd+K на macOS). - У вікні Commit Changes додайте повідомлення для коміту, наприклад, "Initial commit". - Натисніть Commit для збереження змін.
3.3.3. Підключіть локальний репозиторій до GitHub: - Виберіть VCS -> Git -> Remotes. - Натисніть + для додавання нового віддаленого репозиторію. - У полі URL вставте скопійований URL вашого GitHub репозиторію та натисніть OK.
3.4. Опублікуйте проєкт на GitHub: - Виберіть VCS -> Git -> Push. - Перевірте, чи всі файли вибрані для публікації, та натисніть Push.
3.4. Перевірка репозиторію на GitHub
3.4.1. Після успішного виконання команд перейдіть до вашого GitHub репозиторію в браузері.
3.4.2. Переконайтеся, що ваш застосунок з'явився в репозиторії, і всі файли доступні.
3.5. Додавання змін до репозиторію
3.5.1. Внесіть зміни у ваш проєкт, наприклад, змініть текст у файлі розмітки або додайте новий функціонал у MainActivity.
3.5.2. Додайте зміни до індексу Git: - Виберіть VCS -> Git -> Add або використайте сполучення клавіш Ctrl+Alt+A (Cmd+Option+A на macOS).

3.5.3. Зробіть новий коміт із змінами: - Повторіть процес коміту, додаючи повідомлення про зміни.
3.5.4. Знову виберіть VCS -> Git -> Push, щоб публікувати зміни на GitHub.
